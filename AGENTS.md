# Repository Instructions

- Use Conventional Commits with scopes, such as `package`, `react`, `ci`, `release`, `docs`, or `tests`.
- Use `pnpm` for dependency and script commands.
- Run `pnpm run verify` before committing release, package metadata, export map, or runtime changes.
- For publishing, release, versioning, npm provenance, or package tag work, use the repo `publish-package` skill first.
